# Pages API

The Pages API provides CRUD operations for managing pages.

## Endpoints

### `GET /api/pages`

Fetches all pages.

**Response:**

```json
[
  {
    "id": 1,
    "title": "Page 1",
    "slug": "page-1",
    "content": "Content of Page 1",
    "createdAt": "2024-01-01T00:00:00Z",
    "updatedAt": "2024-01-01T00:00:00Z"
  },
  ...
]
```
GET /api/pages/:slug

Fetches a specific page by slug.

Example Request:

GET /api/pages/page-1

Response:
```json
{
  "id": 1,
  "title": "Page 1",
  "slug": "page-1",
  "content": "Content of Page 1",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-01T00:00:00Z"
}
```
PUT /api/pages/:slug

Updates a specific page by slug.

Example Request:

PUT /api/pages/page-1
Content-Type: application/json
```json
{
  "title": "Updated Page 1",
  "content": "Updated content of Page 1"
}
```
Response:
```json
{
  "id": 1,
  "title": "Updated Page 1",
  "slug": "page-1",
  "content": "Updated content of Page 1",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-02T00:00:00Z"
}
```

# Authentication API

The Authentication API allows you to handle admin login and registration.

## Endpoints

### `POST /api/auth/login`

Allows the admin to log in.

**Example Request:**


POST /api/auth/login
Content-Type: application/json
```json
{
  "username": "admin",
  "password": "adminpassword"
}
```
Response:
```json
{
  "token": "jwt-token-here"
}
```
POST /api/auth/register

Allows the admin to register.

Example Request:

POST /api/auth/register
Content-Type: application/json
```json
{
  "username": "newadmin",
  "password": "newadminpassword"
}
```
Response:
```json
{
  "message": "Registration successful"
}
```
